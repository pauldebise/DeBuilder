# Instructions pour OpenCode

## Source de vérité
- Le cahier des charges complet du projet se trouve dans `./cahier_des_charges.md`.
- Toutes les fonctionnalités, architectures et règles de développement doivent se conformer strictly à ce document.

## Règles générales
- **Architecture & Code** : Respecter les choix techniques et l'arborescence définis dans le cahier des charges.
- **Règles Git & Sécurité** : Ne jamais altérer la configuration de suivi Git principale du dépôt sans instruction explicite.
- **Modularité** : Découper le code en modules clairs avec des tests associés.
- **Git** : Créer et mettre à jour les fichiers git (comme .gitignore et .gitattributes par exemple).
